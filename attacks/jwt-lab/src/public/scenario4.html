🔐 Scénario 4 — kid Header Injection

kid Header Injection

Le paramètre kid (Key ID) dans le header JWT indique quelle clé utiliser pour vérifier la signature. Si ce paramètre n'est pas validé, un attaquant peut injecter un chemin arbitraire pour forcer l'utilisation d'une clé vide ou contrôlée.

📚 Comment fonctionne le kid ?

Le serveur utilise un keystore interne :

key-prod→ "s3cr3t_pr0d_k3y"
key-dev→ "devkey"

Le serveur lit le kid du header et cherche la clé correspondante. Si kid = "../../etc/passwd" ou kid = "../nonexistent", le serveur peut charger un fichier vide → signature avec chaîne vide !

// Header JWT manipulé : { "alg": "HS256", "typ": "JWT", "kid": "../../config/empty" ← path traversal ! } // Résultat : serveur utilise clé vide "" pour vérifier // L'attaquant signe son token avec "" → accès admin
