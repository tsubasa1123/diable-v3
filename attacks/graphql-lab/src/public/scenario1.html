🕸️ Scénario 1 — Introspection Attack

GraphQL Introspection

L'introspection GraphQL est une fonctionnalité permettant d'interroger le schéma d'une API pour connaître tous ses types, champs et opérations. Activée en production, elle devient un outil de reconnaissance puissant pour un attaquant.

📚 Qu'est-ce que l'introspection ?

GraphQL expose une requête spéciale __schema qui retourne la structure complète de l'API. Elle révèle des champs sensibles que le développeur croyait "cachés" car non documentés.

query IntrospectionQuery { __schema { types { name fields { name type { name } } } } }

Résultat : l'attaquant découvre des champs comme password, creditCard, salary non documentés mais bien présents dans l'API.
