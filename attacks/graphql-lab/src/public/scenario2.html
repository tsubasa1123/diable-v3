🕸️ Scénario 2 — Data Exfiltration

GraphQL Data Exfiltration

GraphQL permet de demander exactement les champs voulus. Un attaquant peut construire des requêtes pour extraire massivement des données sensibles : mots de passe, salaires, CB, documents confidentiels — en une seule requête.

Cible 1 — Données utilisateurs

Extrayez tous les utilisateurs avec leurs données sensibles :

{ users { id username email role salary password creditCard } }

Cible 2 — Documents confidentiels

Accédez aux documents secrets sans aucune authentification :

{ documents { id title content classification } }

Requête personnalisée — Tout en une fois
