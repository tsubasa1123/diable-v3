🕸️ Scénario 3 — Auth Bypass via Mutation

GraphQL Auth Bypass

La mutation login retourne un token encodé en Base64 contenant le rôle de l'utilisateur. Le serveur fait confiance à ce token sans vérification cryptographique. Il suffit de modifier le rôle dans le token pour obtenir un accès admin.

📚 Le mécanisme vulnérable

Le serveur génère un token ainsi :

// Génération (côté serveur) const token = Buffer.from(JSON.stringify({ id: user.id, role: user.role })).toString('base64'); // Vérification (VULNÉRABLE : pas de signature !) const decoded = JSON.parse(Buffer.from(token, 'base64').toString()); if (decoded.role === 'admin') { /* accès accordé */ }

Sans signature cryptographique, n'importe qui peut modifier le token. C'est exactement la même faille que JWT avec alg:none — mais appliquée à GraphQL.
